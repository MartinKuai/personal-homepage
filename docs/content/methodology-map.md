## AI 工具使用流程

| 字段 | 内容 |
|------|------|
| 方法名称 | AI 工具使用流程 |
| 适用场景 | 用 AI 辅助完成具体任务 |
| 输入 | 任务目标、已有资料 |
| 处理步骤 | 1. 明确目标 → 2. 选择工具 → 3. 编写 prompt → 4. 执行 → 5. 检查结果 |
| 输出 | 任务结果 |
| 注意事项 | 结果需要人工验证，不能直接信任 |
| 当前成熟度 | 使用中，未整理成文档 |
| 是否已在项目中验证 | 是 |

## 项目启动流程

| 字段 | 内容 |
|------|------|
| 方法名称 | 项目启动流程 |
| 适用场景 | 开始一个新项目 |
| 输入 | 模糊的想法 |
| 处理步骤 | 1. 写清楚目标 → 2. 确定技术栈 → 3. 搭建最小框架 → 4. 跑通第一个版本 |
| 输出 | 项目骨架和第一个可运行版本 |
| 注意事项 | 不要一开始就追求完美 |
| 当前成熟度 | 有经验，未整理 |
| 是否已在项目中验证 | 是 |

## 项目验收流程

| 字段 | 内容 |
|------|------|
| 方法名称 | 项目验收流程 |
| 适用场景 | 项目阶段性完成时 |
| 输入 | 项目目标清单 |
| 处理步骤 | 1. 逐项检查 → 2. 记录通过/未通过 → 3. 记录仍需优化的问题 |

| 注意事项 | 要诚实，不要跳过未通过项 |
| 当前成熟度 | 使用中（v0-local-acceptance.md） |
| 是否已在项目中验证 | 是 |

## 内容整理流程

| 字段 | 内容 |
|------|------|
| 方法名称 | 内容整理流程 |
| 适用场景 | 把想法整理成可展示的内容 |
| 输入 | 散乱的笔记、想法、素材 |

| 注意事项 | 不要追求一次写完，可以分阶段 |

| 是否已在项目中验证 | 部分（content 目录已建立） |
