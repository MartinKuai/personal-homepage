## 背景

在前期做过几个小型页面和静态 demo 后，我开始意识到需要一个更长期、可持续维护的内容空间。

之前的页面更多是一次性展示，缺少持续积累的可能性。随着项目和想法增多，我需要一个能承载不同内容类型的主页。

## 目标

这次的目标不是做一个完美的个人网站，而是搭建一个 V0 框架：

- 有基本的内容结构
- 可以用 Markdown 管理内容
- 不依赖复杂后端或外部服务
- 未来可以持续迭代

## 技术选择

选用了 Astro + TypeScript + Tailwind CSS + Markdown。

选择这个组合的原因：

- 静态输出，部署简单
- Markdown 写作体验好
- Tailwind 样式可控
- 不需要数据库和登录

整体风格偏向深色、克制、低饱和。不引入大型 UI 库。

## 做了什么

V0 完成了以下内容：

- 首页基础结构（Hero / About / Focus Areas / Works / Writing / Timeline / Contact）
- Writing 内容系统（列表 + 详情页）
- Project Case 内容系统（列表 + 详情页）
- Timeline 内容集合
- Topics 轻量入口页
- Markdown frontmatter 规范（draft / placeholder / featured 过滤）

## 过程中发现的问题

1. **内容结构需要迭代** - 最初设计的 frontmatter 字段后来发现不够用，需要补充
2. **防御性表达** - 早期写了不少"不是什么"的说明，后来改为正向陈述
3. **中英文混用** - 概念词用英文更自然，操作文案用中文更一致
4. **返回链接** - 二级页面返回首页时需要考虑锚点定位问题

## 当前状态

这个项目还处在早期阶段。内容数量较少，结构也在持续验证中。

首页展示方式、内容组织、细节交互都可能在后续调整。
